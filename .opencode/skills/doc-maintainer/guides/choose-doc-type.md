# 选择文档类型

先判断读者是谁，再决定文档归属。

## 归类规则

- 面向产品使用者、部署执行者、首次接触项目的人，归为用户文档
- 面向仓库贡献者、维护者、Agent 或开发协作者，归为开发文档
- 同时覆盖两类受众时，拆成两份文档互相链接，不要混写

## 用户文档适用内容

- 项目介绍
- 快速开始
- 使用方式
- 部署入口
- 常见问题

## 开发文档适用内容

- 本地开发
- 架构说明
- 代码规范
- git 工作流
- Agent 使用规范

## 强制判断点

- 如果文档需要告诉读者如何修改代码、提交变更或使用项目 skill，则它是开发文档
- 如果文档需要告诉读者如何理解项目、快速上手或进入部署流程，则它是用户文档

## 产出要求

- 选定类型后，再进入对应 guide
- 无法单独归类时，先拆分受众，再为两类文档建立跳转链接
