# 新增用户文档

用户文档服务于项目使用者与首次接触项目的人，优先解决“这是什么、怎么开始、去哪里继续”。

## 根用户文档必备内容

根用户文档必须显式包含以下四项，缺一不可：

- 项目介绍
- 快速开始
- 开发文档链接
- 部署文档链接

## 推荐结构

1. 项目介绍
2. 快速开始
3. 核心使用路径或主要能力
4. 开发文档链接
5. 部署文档链接
6. 常见问题或补充说明

## 写作要求

- 默认使用中文
- 面向非开发者也能理解的表述，避免把内部实现细节塞进主流程
- 如果引用开发侧约束，只给出链接，不在用户文档展开实现细则
- 入口文档必须让读者在最短路径内找到开始方式和后续文档

## 禁止事项

- 不省略根用户文档四项必备内容
- 不把代码规范、git 工作流、Agent 使用规范写成用户主文档主体
- 不假定读者已经了解仓库结构或内部术语
