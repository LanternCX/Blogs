# 文档风格参考

## 基本风格

- 默认简洁、直接、可执行
- 标题体现读者任务，不写空泛口号
- 列表优先于大段叙述
- 术语前后一致，避免同义混用

## 面向用户文档

- 优先写读者能立即采取的动作
- 减少内部实现细节
- 用清楚的链接文案指向开发文档和部署文档

## 面向开发文档

- 优先写规则入口、协作路径与目录位置
- 对代码规范、git 工作流、Agent 使用规范使用明确标题
- 提到项目 skill 时，直接写 `.opencode/skills/`

## 不推荐写法

- 只有背景，没有操作入口
- 只说“见相关规范”，不提供链接或位置
- 用户文档与开发文档混用同一套话术，导致受众不清
