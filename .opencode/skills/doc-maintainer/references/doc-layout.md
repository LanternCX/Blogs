# 文档布局参考

## 总原则

- 先给入口，再给细节
- 先回答读者当前要做什么，再展开背景
- 用户文档与开发文档分层维护，交叉处用链接连接

## 用户文档布局

根用户文档至少应按以下顺序组织：

1. 项目介绍
2. 快速开始
3. 关键使用信息
4. 开发文档链接
5. 部署文档链接

可选补充：常见问题、进阶用法、术语说明。

## 开发文档布局

开发文档至少应覆盖：

1. 开发入口或环境准备
2. 仓库结构或关键模块说明
3. 代码规范
4. git 工作流
5. Agent 使用规范
6. 项目 skill 位置：`.opencode/skills/`

可选补充：测试策略、发布流程、故障排查。

## 链接布局

- 用户文档链接到开发文档时，用明确标题，不用“更多信息”这类弱文本
- 开发文档引用规则时，优先链接现有 skill 或规范入口
- 同一规则只有一个主入口，其他文档只做导航
