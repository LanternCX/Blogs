# code review 检查清单

## 架构与模块

- 模块职责是否单一且可清晰命名
- 是否存在跨层直连、循环依赖或双向耦合
- 横切能力是否统一收敛, 没有散落到多个业务模块

## 注释与文档

- 公共接口与关键边界是否使用 Doxygen 风格文档注释
- 是否默认使用 `@brief`, 并仅在必要时补 `@param`、`@return`、`@note`、`@warning`
- 注释是否为中文
- 注释与文档字符串是否使用半角标点, 且标点后有一个空格
- 单行注释行尾是否没有句号、逗号、分号、冒号等收尾标点

## 持久化与路径

- 需要持久化的文件是否全部落在程序运行目录
- 是否存在写入系统目录或用户全局目录的默认行为
- 路径规则是否由统一层管理, 而不是业务模块各自处理

## 日志

- 日志系统是否同时支持 stdout 与可持久化滚动日志
- 两种输出是否共享统一日志语义与级别
- 日志文件是否写入程序运行目录
- 关键错误路径与状态变化是否有足够日志支撑排查
