# 补充注释

## 目标

注释用于解释职责、约束、前置条件与不直观决策, 不重复代码字面含义。

## 操作步骤

1. 对公共接口、核心类型、关键流程优先补文档注释
2. 到 `references/comment-style.md` 选择 Doxygen 标签与标点写法, 不在此处重复定义格式细则
3. 只为复杂分支、边界条件、约束来源写行内注释, 不解释基础语法

## 快速判断

- 能靠更清晰命名解决的, 优先改命名
- 需要说明边界、原因、风险时, 写注释
- 只是复述代码在做什么时, 不写注释

## 常见反模式

- 混用普通块注释与 Doxygen 文档注释描述公共接口
- 明明已经进入 `references/comment-style.md`, 仍然按个人习惯另起一套格式
