# 重构模块

## 目标

重构时优先收敛职责、切断无效依赖与隐藏实现细节, 保持对外语义稳定。

## 操作步骤

1. 先识别当前模块承担的职责清单, 找出不属于该模块的能力
2. 以调用方向为线索整理依赖, 去掉循环依赖、双向感知与跨层直连
3. 把易变实现抽离到内部私有结构或独立协作者, 保持对外接口稳定
4. 需要补注释时, 回到 `guides/add-comments.md` 与 `references/comment-style.md`
5. 涉及落盘路径时, 回到 `references/persistence-rules.md` 核对程序运行目录约束
6. 涉及日志链路时, 回到 `guides/add-logging.md` 与 `references/logging-rules.md`

## 重构判断

- 如果问题来自职责混杂, 先拆模块再改实现
- 如果问题来自依赖错位, 先调依赖方向再谈复用
- 如果问题来自横切能力散落, 先统一注入点再继续扩展

## 常见反模式

- 只改命名不改边界, 让耦合继续存在
- 为了复用把不同场景硬塞进一个抽象层
- 重构后把注释、日志、持久化规则留成历史包袱
