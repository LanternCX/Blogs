# 增加日志

## 目标

日志用于支撑运行期观测、问题定位与长期追踪, 必须兼顾实时输出与可持久化留存。

## 操作步骤

1. 先确定要记录的事件边界, 只覆盖关键状态变化、失败路径与重要业务节点
2. 统一通过日志组件输出, 不在业务代码中随意拼接多套日志通道
3. 到 `references/logging-rules.md` 确认 stdout 与可持久化滚动日志的双通道要求
4. 到 `references/persistence-rules.md` 确认日志文件位于程序运行目录下的约定位置
5. 日志内容优先记录能帮助定位问题的上下文, 避免无意义重复噪音
6. 对错误、重试、降级、外部依赖失败等路径必须有可检索日志

## 落地提醒

- stdout 与滚动日志的具体约束以 `references/logging-rules.md` 为准
- 日志文件路径以 `references/persistence-rules.md` 为准

## 常见反模式

- 只写控制台日志, 导致运行后无法追溯
- 只写文件日志, 导致实时观测与平台采集缺失
- 在多个模块里各自维护日志文件路径与滚动策略
