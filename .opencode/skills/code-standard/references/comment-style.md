# 注释规范

## 硬规则

- 本项目注释统一使用中文
- 文档注释统一采用 Doxygen 风格
- 文档注释默认使用 `@brief`
- 仅在需要说明参数、返回值、注意事项、风险时补 `@param`、`@return`、`@note`、`@warning`
- 注释与文档字符串中的标点统一使用半角符号
- 标点后必须跟一个空格
- 单行注释行尾禁止使用句号、逗号、分号、冒号等收尾标点

## 适用范围

- 公共函数、公共方法、导出类型、核心模块入口使用文档注释
- 复杂条件分支、性能权衡、兼容性处理使用最小必要行内注释
- 纯赋值、简单判断、直接调用等显然逻辑不写注释

## 推荐写法

- `@brief` 先说职责, 再说边界
- `@param` 只解释调用方从名称看不出的约束
- `@return` 只解释返回语义或失败约定
- `@note` 用于补充实现选择、兼容性或上下文限制
- `@warning` 用于提示高风险副作用、数据覆盖、性能代价等

## 禁止写法

- 用普通块注释替代公共接口文档注释
- 在中文句子中混入全角 `，` `。` `：` `；`
- 标点后直接接中文或英文字符而不留空格
- 单行注释以 `。` `,` `;` `:` 等符号收尾
