# 日志规则

## 硬规则

- 日志系统必须同时支持 stdout 与可持久化滚动日志
- 两种输出共享同一套事件语义与日志级别
- 可持久化滚动日志文件必须写入程序运行目录
- 日志能力属于基础设施能力, 不得在业务模块中分散实现

## 设计要求

- stdout 用于实时查看、容器采集与开发调试
- 滚动日志用于持久化留存、问题回溯与离线分析
- 关键事件至少包含事件名称、结果、必要上下文与错误信息
- 日志格式可以结构化, 但不能牺牲可读性与检索性

## 接入要求

- 模块只依赖统一日志入口, 不感知底层输出细节
- 初始化阶段一次性装配 stdout 与文件滚动策略
- 业务逻辑只决定记什么, 不决定日志写到哪里

## 禁止写法

- 仅输出 stdout
- 仅输出文件日志
- 在不同模块中各自创建不同日志目录或滚动策略
- 将日志文件落到系统目录或用户全局目录
