# 持久化规则

## 硬规则

- 需要持久化的文件一律写到程序运行目录
- 禁止写入系统目录, 例如 `/etc`、`/var/lib`、`/usr/local/share`
- 禁止默认写入用户全局目录, 例如用户 home 下与当前程序无关的散落路径
- 需要持久化的相对路径必须由程序运行目录推导, 不依赖机器特定绝对路径

## 适用对象

- 本地数据库文件
- 缓存快照、索引文件、状态文件
- 日志文件、导出文件、运行期生成但需要保留的持久化资产

## 落地要求

- 路径策略可配置, 但默认值必须落在程序运行目录下
- 模块内部不要各自发明路径规则, 统一由配置层或基础设施层下发
- 创建目录、检测权限、清理历史文件时, 也必须围绕程序运行目录展开

## 风险提示

- 写入系统目录会带来权限差异、部署不一致与清理困难
- 写入用户全局目录会让数据位置不可预期, 降低可移植性
