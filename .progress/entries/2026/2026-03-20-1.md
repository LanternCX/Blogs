## Date
2026-03-20

## Title
完成 Hexo 基础骨架与样本迁移闭环

## Background / Issue
需要把现有 Markdown 仓库迁到标准 Hexo 结构, 先完成第一阶段的基础骨架、迁移规则、抽样迁移和渲染验证, 同时保持 git flow、TDD 和不自动 commit 的项目约束。

## Actions / Outcome
- Approach 1: 先落地 Hexo 站点骨架与最小 README -> 建立了可运行的 npm/Hexo 入口
- Approach 2: 先写迁移规则失败测试, 再逐模块实现 -> 完成标题提取、Git 时间归一化、页面/文章分流、front matter 渲染、路径映射与样本集合
- Approach 3: 实现抽样迁移脚本并生成映射文档 -> 成功迁移 3 篇文章与 2 个页面, 产出 `docs/migration/hexo-sample-mapping.md`
- Approach 4: 以真实渲染而非仅 HTTP 200 作为闭环标准 -> 补齐 `hexo-renderer-marked`、`hexo-generator-index`、`hexo-theme-landscape`、`hexo-renderer-ejs` 等最小依赖, 打通首页、页面和文章详情页渲染
- Final approach: 用子代理分 Task 1-6 执行并做双重审查 -> 当前 `npm test && npm run migrate:samples && npm run hexo:clean && npm run hexo:generate` 通过, 且手工附录不会再被迁移脚本覆盖

## Lessons / Refinements
- Hexo 第一阶段不能只补 `hexo` 和 `hexo-cli`, 还要检查 generator、theme、renderer 是否齐全, 否则容易出现 200 但模板未真正渲染
- 迁移说明文档如果既有自动生成区又有手工维护区, 必须在脚本层显式保留手工附录, 否则 README 引导命令会破坏文档完整性
- 渲染验证不能只看状态码, 还要检查输出 HTML 中不存在原始模板标记, 并存在正向内容信号
